KEY OBSERVATIONS
================
1. The build is REAL, not faked:
   - "Compiling hello v0.1.0 (/workspace)" appears on every run after an edit.
   - Wall-clock build times vary (32.81s, 8m 19s, 2.07s, 2.02s) — characteristic
     of a real rustc invocation inside CheerpX, not a canned string.
   - The reporter's own edits ("! 2", "! 3", "! 5") show up in stdout, which is
     only possible if the source is actually recompiled and the binary actually run.

2. The smell that triggered the report — the branding lines:
   - First run prints  : "This binary was compiled inside CheerpX."
   - Edited runs print : "Compiled by Rust inside the browser via CheerpX."
   Two DIFFERENT marketing-style lines for the same project. The first comes from
   the prebuilt binary baked by web/disk/Dockerfile.disk; the later ones come from
   the JS workspace seed in web/glue/workspace-fs.js. They were never the same.
   That divergence is what made the output "look like something we added in code."
